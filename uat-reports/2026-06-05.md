## Pages-adapted method (why no `-test` worker)

The canonical `static-site-form-uat` deploys a `{name}-test` worker **without** `TURNSTILE_SECRET_KEY`,
relying on the form code degrading gracefully so Playwright can POST. This site's Phase-4 hardening is
**fail-secure** — a missing Turnstile secret returns **500**, not an open endpoint — so that trick would
yield a dead endpoint, not a testable one. Instead the harness runs `wrangler pages dev` locally with
`.dev.vars` carrying Turnstile's **always-pass TEST secret** (`1x0000…AA`): the secret is *present*
(fail-secure satisfied) but accepts any token, so automated POSTs reach the real GHL + n8n path.
Production secrets were never touched.
